## 方案 ① Markdown 文件驱动（最主流）

> 每篇文章就是一个 `.md` 文件，丢进文件夹就行

### 目录结构：
```
project/
├── content/
│   └── posts/
│       ├── 2024-01-01-my-first-post.md
│       ├── 2024-01-15-second-post.md
│       └── 2024-02-01-third-post.md
├── src/
└── ...
```

### 文章长这样：
```markdown
---
title: "我的第一篇博客"
date: "2024-01-01"
tags: ["生活", "技术"]
summary: "这是文章摘要"
cover: "/images/cover1.jpg"
---

# 正文开始

这里写正文，支持所有 Markdown 语法...

## 代码高亮
​```javascript
console.log('Hello World')
​```

## 图片
![描述](./images/photo.jpg)

## 列表
- 项目1
- 项目2
```

### 写作流程：
```
新建 .md 文件 → 写内容 → git push → 自动部署 ✅
```

**不需要碰任何代码，只管写 `.md` 文件！**

## 方案 ② Headless CMS（🌟 最推荐）

> 给你一个**在线后台**，像 WordPress 一样写文章

### 推荐几个免费的：

### 🥇 Sanity（最推荐）
```
特点：
- 可视化编辑器，实时预览
- 免费额度很够用（个人博客完全够）
- 和 Next.js 完美配合
```

### 🥈 Notion 当 CMS
```
特点：
- 你可能已经在用 Notion 了
- 直接在 Notion 里写，自动同步到博客
- 最省事！
```

### 🥉 Contentful / Strapi
```
特点：
- 企业级方案
- 功能强大但学习成本稍高
```

### Sanity 的体验是这样的：

```
打开浏览器 → 登录后台 → 点"新建文章" → 写 → 发布 ✅
```

**完全不碰代码，手机上都能写！**

---

## 方案 ③ 自建后台（适合想练手）

> 自己做一个 Admin 管理面板

```
需要做的事情：
├── 1. 搭建数据库（MongoDB / PostgreSQL）
├── 2. 写 API 接口（CRUD）
├── 3. 做后台页面 /admin
│   ├── 登录认证
│   ├── 文章列表
│   ├── 富文本编辑器（TipTap / Editor.js）
│   └── 图片上传
└── 4. 前台读取数据展示
```

**工作量最大，但自由度最高。**

---

## 方案 ④ GitHub 在线编辑（最懒人）

> 直接在 GitHub 网页上编辑 Markdown 文件

```
打开 GitHub 仓库 → 进入 content/posts/ → 
新建文件 → 写 Markdown → Commit → 自动部署 ✅
```

配合 Vercel/Netlify 自动部署，push 了就更新。

---

## 🎯 我的推荐

根据你的情况（不想碰 VSCode，想在网站上操作）：

```
如果你想最快搞定 → 方案 ① Markdown + GitHub 在线编辑
如果你想体验最好 → 方案 ② Sanity / Notion CMS
如果你想学习成长 → 方案 ③ 自建后台
```
